## 🔄 核心流程图

### 1️⃣ 游戏开场流程

```
用户打开 index.html
       ↓
设备检测 (PC/Mobile)
       ↓
  ┌────┴────┐
  ↓         ↓
PC 端     移动端
  ↓         ↓
index.html  mobile.html
  ↓         ↓
  └────┬────┘
       ↓
第一步：封面页面 (step1Page)
  - 夏威夷风景背景
  - 装饰图片
  - 点击右下角进入下一步
       ↓
第二步：ALOHA 欢迎页 (step1_5Page)
  - 粉色背景 + 星空
  - 打字效果欢迎文字
  - 点击任意位置继续
       ↓
第三步：机票输入页 (step2Page)
  - 完整机票样式
  - 输入玩家姓名
  - 填写航班信息
       ↓
第四步：动画消散 (step3Page)
  - 机票化作星光粒子
  - 纸飞机飞出动画
  - "旅程开始"提示
       ↓
跳转到 game.html?name=玩家姓名
       ↓
游戏正式开始！
```

### 2️⃣ 游戏主循环流程

```
game.html 加载
       ↓
entry.js (入口模块)
       ↓
initGame() 初始化
  - 加载 gameState
  - 预加载图片
  - 检查 API Key
       ↓
startCustomsScene() (海关场景)
       ↓
┌─────────────────────────────────┐
│      游戏主循环 (Game Loop)      │
│                                 │
│  1. 显示场景旁白                 │
│  2. 生成 NPC 对话 (AI)            │
│  3. 等待玩家输入                 │
│  4. handleInput() 处理输入       │
│     ├─ 显示玩家消息              │
│     ├─ 保存到对话历史            │
│     ├─ 调用 AI 获取回复           │
│     ├─ 翻译成中文                │
│     ├─ 显示 NPC 回复              │
│     ├─ 生成文化提示              │
│     └─ 检查剧情进度              │
│  5. checkStoryProgress()         │
│     ├─ 海关阶段 → 遇到导游       │
│     ├─ 导游阶段 → 机场邂逅       │
│     ├─ 邂逅阶段 → 触发事件       │
│     └─ 完成阶段 → 切换地点       │
│  6. 重复步骤 3-5                 │
└─────────────────────────────────┘
       ↓
章节切换 (ChapterManager)
  - 保存当前对话历史
  - 加载新章节对话
  - 更新剧情状态
       ↓
游戏结束/继续下一章
```

### 3️⃣ AI 调用流程

```
玩家输入消息
       ↓
handleInput()
       ↓
getAIResponse(prompt, npcName, history)
       ↓
GameCore.api.getAIResponse()
       ↓
构造 API 请求
  - system: NPC 人设 + 玩家上下文
  - user: 玩家输入 + 对话历史
  - temperature: 0.7
       ↓
调用 DeepSeek API
       ↓
接收响应
       ↓
parseJSONResponse() (如果需要)
       ↓
返回 AI 回复 (英文)
       ↓
translateToChinese()
       ↓
GameCore.api.translateToChinese()
       ↓
构造翻译请求
  - system: "将以下英文翻译成中文"
  - user: AI 回复内容
       ↓
调用 DeepSeek API
       ↓
返回中文翻译
       ↓
displayNPCMessage(npc, english, chinese)
       ↓
UI 显示 (双语消息)
```

## 🌟 INFP 架构特点分析

### ✅ 优势

1. **高度模块化** - 虽然"没规划"，但模块划分清晰
2. **功能完整** - AI 对话、章节系统、音频管理、成就系统应有尽有
3. **文档丰富** - 各种指南、设计文档、故事大纲
4. **可扩展性强** - 每个模块相对独立，容易添加新功能
5. **创意驱动** - 功能设计充满想象力和浪漫色彩

### ⚠️ 问题

1. **全局状态分散** - gameState 在多处使用，缺乏统一管理
2. **模块耦合度高** - game.js、entry.js、core.js 互相依赖
3. **命名不一致** - 有些用驼峰，有些用下划线
4. **重复代码** - AI 调用、错误处理有重复
5. **缺少类型系统** - TypeScript 只有空壳
