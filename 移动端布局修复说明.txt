📱 移动端布局问题修复 📱
========================================

修复时间：2026-04-11
修复内容：解决屏幕缩小时的布局问题

========================================
🐛 修复的问题
========================================

【问题 1：发送键和录音键被吞掉】
-----------------------------
原因：输入框太宽，挤占了按钮空间
修复：
✅ 添加 input-wrapper 包装器
✅ 设置 flex 布局，允许缩小
✅ 按钮设置 flex-shrink: 0（不缩小）
✅ 输入框设置 min-width: 0（允许缩小）
✅ 添加 gap: 8px 间距

【问题 2：顶框地点标签溢出】
-----------------------------
原因：地点文字太长，屏幕窄时溢出
修复：
✅ 减少 max-width: 80% → 70%
✅ 添加 overflow: hidden
✅ 添加 text-overflow: ellipsis
✅ 文字设置 max-width: 150px
✅ emoji 设置 flex-shrink: 0（不缩小）

========================================
🔧 技术细节
========================================

【输入区域重构】
-----------------------------
原结构：
.input-container
  └─ div (inline style)
      ├─ .input-area
      ├─ #voiceInputBtn (inline style)
      └─ button (inline style)

新结构：
.input-container
  └─ .input-wrapper
      ├─ .input-area
      ├─ .icon-btn (#voiceInputBtn)
      └─ .send-btn.mobile

优势：
- 移除 inline style，使用 CSS 类
- 更好的响应式控制
- 代码更清晰易维护

【布局优化】
-----------------------------
关键 CSS 属性：

flex-shrink: 0  /* 不缩小，保持固定大小 */
min-width: 0    /* 允许缩小到内容大小 */
flex: 1         /* 占满剩余空间 */
overflow: hidden /* 隐藏溢出内容 */
text-overflow: ellipsis /* 显示省略号 */

【响应式断点】
-----------------------------
@media (max-width: 375px)   /* 小屏手机 */
@media (max-width: 350px)   /* 超小屏幕 */

小屏幕适配策略：
- 地点文字进一步缩小
- 减少 padding 和 gap
- 调整 max-width 百分比
