# 环境变量说明

## 本地开发设置

1. 复制 `.env.example` 为 `.env`
   ```bash
   cp .env.example .env
   ```

2. 编辑 `.env` 文件，填入你的 API Key：
   ```
   VITE_QIANFAN_API_KEY=你的实际 API_Key
   ```

3. `.env` 文件不会被提交到 Git（已在 .gitignore 中）

## GitHub Actions 设置

如果要在 GitHub Actions 中使用，需要设置 Secrets：

1. 进入 GitHub 仓库 → Settings → Secrets and variables → Actions
2. 点击 "New repository secret"
3. 添加以下 Secrets：
   - `QIANFAN_API_KEY`: 你的 API Key
   - `QIANFAN_SECRET_KEY`: 你的 Secret Key（可选）

## 本地存储方式（当前使用）

项目目前使用 localStorage 存储 API Key：
- 用户可以在设置界面输入自己的 API Key
- API Key 保存在浏览器本地，不会上传到服务器
- 这是最安全的方式

## 安全建议

✅ 推荐：使用 localStorage（当前方式）
- 用户自己管理 API Key
- 不硬编码在代码中
- 不提交到版本控制

⚠️ 不推荐：硬编码在代码中
- 容易泄露
- 需要手动管理
- 不适合公开仓库
