# 🧪 章节切换系统测试指南

## 📋 问题背景

章节切换功能很难测试，因为需要：
1. 完整走完游戏流程（海关 → 遇到导游 → 选择目的地）
2. 等待对话生成
3. 无法快速验证不同场景

## ✅ 解决方案

添加了**一键测试按钮**，可以在章节面板中快速测试章节切换功能。

---

## 🚀 使用方法

### 方法 1：使用测试按钮（推荐）

1. **打开游戏**
   - 访问 `mobile.html`
   - 输入名字开始游戏

2. **打开章节面板**
   - 从屏幕**右边缘向左滑动**
   - 或者点击功能栏的"📖 章节"按钮

3. **点击测试按钮**
   - 在章节面板底部找到"🧪 开发测试"区域
   - 点击"🚀 一键测试章节切换"按钮

4. **观察测试结果**
   - 系统会自动完成引子章节
   - 播放章节切换动画
   - 显示测试报告
   - 提供后续测试选项

### 方法 2：使用控制台命令

打开浏览器开发者工具（F12），在控制台输入：

```javascript
testChapterSwitchSimple()
```

---

## 📊 测试内容

测试会自动验证以下功能：

### ✅ 1. 章节进度更新
- 引子章节任务完成
- DAY 1 解锁

### ✅ 2. 章节切换动画
- 全屏过渡动画播放
- 动画时长 2 秒

### ✅ 3. 回调函数触发
- 动画完成后执行回调
- 不再依赖硬编码的 setTimeout

### ✅ 4. 对话历史保存
- 保存 HTML 内容
- 保存 conversationHistory（AI 对话上下文）

### ✅ 5. 章节状态显示
- 当前章节索引
- 已保存的对话历史数量
- conversationHistory 长度

---

## 🔄 测试场景

测试成功后会显示两个选项：

### 选项 A：🔙 切换回引子章节
- 测试从 DAY 1 切换回引子章节
- 验证对话历史是否正确恢复
- 检查 gameState.conversationHistory 是否还原

### 选项 B：🏖️ 开始 DAY 1 剧情
- 测试正式进入 DAY 1 流程
- 验证海滩场景是否正确显示
- 检查地点系统是否正常工作

## 🐛 调试技巧

### 查看日志

打开浏览器控制台（F12），可以看到详细的调试信息：

```
=== 🧪 快速测试章节切换系统 ===
1️⃣ 模拟引子章节完成...
✅ 引子章节完成：true
2️⃣ 切换到 DAY 1...
🔄 开始切换到章节 1...
✅ 章节 引子 的对话已保存 (HTML + conversationHistory)
✅ 章节切换动画完成回调触发
```

### 检查章节状态

在控制台输入：

```javascript
// 查看当前章节索引
ChapterManager.currentChapter

// 查看已保存的对话历史
ChapterManager.dialogueHistories

// 查看已保存的 conversationHistory
ChapterManager.conversationHistories

// 查看章节列表状态
gameChapters
```

### 手动测试对话历史恢复

1. 运行 `testChapterSwitchSimple()`
2. 与 NPC 进行几轮对话
3. 左滑打开章节面板
4. 点击"引子"章节卡片
5. 检查对话是否完整恢复
6. 尝试继续对话，验证 AI 是否记得之前的内容

## 🎯 常见问题

### Q: 点击测试按钮没反应？
**A:** 检查浏览器控制台是否有错误信息。确保所有 JS 文件已加载。

### Q: 动画播放了但没有测试报告？
**A:** 可能是回调函数未正确执行。检查 `switchToChapter` 函数定义。

### Q: 切换章节后对话历史丢失？
**A:** 检查 `ChapterManager.conversationHistories` 是否正确保存和恢复。

### Q: 测试按钮在哪里？
**A:** 打开章节面板（右滑手势），滚动到底部，在"🧪 开发测试"区域。
